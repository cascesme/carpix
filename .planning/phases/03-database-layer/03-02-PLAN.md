- SELECT: async with self._engine.connect() as conn — use engine.connect(), not engine.begin()

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → CacheRepository | Composite key inputs (brand_key, model_key, year) arrive from the URL path in future Phase 5/6; they cross the trust boundary into SQL. |
| FastAPI app → PostgreSQL | All DB communication goes through asyncpg via SQLAlchemy; connection credentials come from DATABASE_URL env var. |
| alembic/env.py → os.environ | DATABASE_URL must be set; missing env var causes KeyError at migration time (not a security risk, but an availability risk). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Tampering | cache_repository.py SQL | mitigate | All SQL uses text() with named params (:brand_key, :model_key, :year, :local_path, :source_url, :file_title) — never f-string interpolation. ASVS V5 requirement satisfied. |
| T-03-04 | Information Disclosure | create_async_engine() | mitigate | echo=False is the SQLAlchemy default; do not set echo=True. SQL with param values must never appear in logs. Verify by not passing echo=True to create_async_engine(). |
| T-03-05 | Denial of Service | AsyncEngine pool exhaustion | mitigate | SQLAlchemy default pool_size=5, max_overflow=10 is sufficient for internal microservice traffic. Pool is lifespan-scoped (shared across requests). No per-request engine creation. |
| T-03-06 | Information Disclosure | DATABASE_URL in env | accept | DATABASE_URL contains DB credentials. Acceptable: internal Docker network, no external exposure. Phase 7 (containerization) will manage secrets via Docker env vars. |
| T-03-07 | Elevation of Privilege | alembic migrations at startup | accept | Alembic runs as the DB user in DATABASE_URL. That user must have CREATE TABLE permission. For production, a dedicated migration user is appropriate — deferred to Phase 7 ops configuration. |
| T-03-SC | Tampering | pip installs (sqlalchemy, alembic) | accept | Package Legitimacy Audit in RESEARCH.md: all packages are [VERIFIED] with slopcheck OK, established publishers, high download counts — no blocking checkpoint required. |
</threat_model>
