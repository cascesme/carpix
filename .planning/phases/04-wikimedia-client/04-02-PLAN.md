<interfaces>
<!-- Contracts the implementation must satisfy -->

WikimediaClient public interface (from RED tests in test_wikimedia.py):
  find_jpeg_url(brand: str, model: str, year: int) -> str | None
    - Primary query: f"{year} {brand} {model}"
    - Fallback query: f"{brand} {model}" (called only if primary returns None)
    - Returns thumburl string exactly as received from Wikimedia API
    - Returns None when both queries produce no JPEG candidate
    - Never raises on empty results (only raises on HTTP errors via raise_for_status)

WikimediaClient._search_first_jpeg internal contract:
  _search_first_jpeg(query: str) -> str | None
    - Calls GET https://commons.wikimedia.org/w/api.php with params dict (see RESEARCH.md API Reference)
    - Calls response.raise_for_status() before parsing
    - Parses: data.get("query", {}).get("pages", {}) — two chained .get() with empty-dict defaults
    - Sorts pages by p.get("index", 999) ascending
    - Iterates sorted pages; for each page: checks imageinfo[0]["mime"] == "image/jpeg"
    - Returns str(info["thumburl"]) for the first JPEG match
    - Returns None if no JPEG found

Wikimedia API params (all string values):
  action="query", generator="search", gsrsearch=<query>, gsrnamespace="6",
  gsrlimit="10", prop="imageinfo", iiprop="url|mime", iiurlwidth="800", format="json"

          self._engine = engine

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WikimediaClient → commons.wikimedia.org | Outbound HTTP; response data is untrusted JSON from external service |
| Response JSON → Python dict | data.get() chain with empty-dict defaults prevents KeyError on missing keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-S | Spoofing | Wikimedia API response | accept | Response authenticity is Wikimedia's responsibility; HTTPS transport ensures no MITM; this service has no auth context to protect |
| T-04-02-T | Tampering | gsrsearch param (brand/model input) | mitigate | httpx params= dict URL-encodes all values automatically; brand/model strings never interpolated directly into URL string |
| T-04-02-E | Elevation of Privilege | thumburl from API response (SSRF vector) | accept | thumburl is only returned to Phase 5 caller as a string; Phase 5 is responsible for validating upload.wikimedia.org host before downloading — noted as cross-phase concern in RESEARCH.md |
| T-04-02-D | Denial of Service | Wikimedia API timeout | mitigate | httpx.Timeout(10.0) set on AsyncClient at construction time in Phase 5 lifespan; if timeout fires, httpx.ReadTimeout propagates to caller |
| T-04-02-ID | Information Disclosure | User-Agent header | accept | _USER_AGENT contains only app name and public repo URL; no secrets |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan; httpx move already validated in Plan 01 |
</threat_model>
