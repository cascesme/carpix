        - carpix_images_data:/images

          condition: service_healthy

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → compose network | port 8000 exposed to host; DATABASE_URL contains credentials |
| carpix-images → postgres | Internal compose network; credentials in environment block |
| volume mount → host | carpix_images_data and postgres_data are Docker managed volumes on host FS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Information Disclosure | DATABASE_URL in docker-compose.yml | accept | Development credentials only; production secrets must be injected via env vars or secrets manager (OPS scope) |
| T-07-06 | Tampering | postgres:16-alpine base image | accept | Official postgres image; production should pin digest (v2 scope) |
| T-07-07 | Denial of Service | Postgres not ready at carpix-images startup | mitigate | service_healthy condition with pg_isready healthcheck prevents premature app start |
| T-07-08 | Elevation of Privilege | alembic in command override runs as appuser | accept | appuser has no write access to /app source — only /images and .venv; Alembic writes only to DB |
| T-07-09 | Information Disclosure | Postgres port not exposed to host | mitigate | No ports: directive on postgres service — DB only reachable on internal compose network |
| T-07-SC | Tampering | postgres:16-alpine package installs | accept | Alpine base; no user-installed packages in this plan |
</threat_model>
