    Create Dockerfile at project root using two stages: `builder` and `runtime`.

    BUILDER STAGE:
    - Base: python:3.12-slim
    - Install uv via `pip install uv` (no curl/apt needed; pip is available in the slim image)
    - Set WORKDIR /app
    - Copy pyproject.toml and uv.lock first (layer caching)
    - Run `uv sync --no-dev --frozen` to install production-only deps into .venv inside /app
    - Copy full source after deps are resolved: src/, alembic/, alembic.ini

    RUNTIME STAGE:
    - Base: python:3.12-slim (same base for glibc compat)
    - Set WORKDIR /app
    - Create non-root user and group: `RUN groupadd -r appuser && useradd -r -g appuser appuser`
    - Copy venv from builder: `COPY --from=builder /app/.venv /app/.venv`
    - Copy application source from builder: `COPY --from=builder /app/src /app/src`
    - Copy alembic files from builder: `COPY --from=builder /app/alembic /app/alembic`
    - Copy alembic.ini from builder: `COPY --from=builder /app/alembic.ini /app/alembic.ini`
    - Create /images directory and set ownership: `RUN mkdir -p /images && chown appuser:appuser /images`
    - Set PATH to include venv bin: `ENV PATH="/app/.venv/bin:$PATH"`
    - Set PYTHONPATH to include src: `ENV PYTHONPATH="/app/src"`
    - Switch to non-root user: `USER appuser`
    - Expose port 8000
    - CMD: `["uvicorn", "carpix_images.main:app", "--host", "0.0.0.0", "--port", "8000"]`

    Do NOT run Alembic migrations inside the Dockerfile CMD — migrations run as a separate step in docker-compose (plan 02).
    Do NOT install curl, wget, or any apt packages beyond the slim base defaults.
    Do NOT use ADD; use COPY only.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| base image → builder | python:3.12-slim from Docker Hub; supply-chain risk if image is tampered |
| builder → runtime | COPY --from=builder copies compiled bytecode and venv — no internet access at runtime stage |
| container → host | /images volume mount; appuser inside container writes to host volume |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Elevation of Privilege | Container runtime | mitigate | Non-root appuser via `USER appuser` in runtime stage; UID has no sudo, no capabilities |
| T-07-02 | Tampering | Base image (python:3.12-slim) | accept | Official Docker Hub image; digest pinning deferred (v2 scope); image is rebuilt on each CI push |
| T-07-03 | Tampering | uv install in builder | accept | `uv sync --frozen` uses uv.lock pinned hashes; lock file is committed |
| T-07-04 | Information Disclosure | .env file in image | mitigate | .dockerignore excludes .env; DATABASE_URL injected at runtime via docker-compose env_file or env vars |
| T-07-SC | Tampering | pip install uv in builder | accept | uv is installed via official pip; builder stage discarded — only venv artifacts copied to runtime |
</threat_model>
