<objective>
Fix 403 Forbidden errors when downloading images from the Wikimedia CDN.

Root cause: the shared `httpx.AsyncClient` in `main.py` is created with no default headers.
`WikimediaClient` works around this by passing `headers={"User-Agent": ...}` on every API
call, but `ImageService._http_client.get(url, ...)` (the actual image download) sends no
User-Agent header, causing `upload.wikimedia.org` to return 403.

Fix: add `headers={"User-Agent": "carpix-images/0.1 (https://github.com/user/carpix)"}` to
the `AsyncClient` constructor in `main.py`. The header then propagates automatically to every
request made through that shared client, including the CDN download in `image_service.py:55`.

Also update test helpers that construct bare `httpx.AsyncClient()` instances so they match
production configuration and tests remain an honest representation of the running service.
</objective>
